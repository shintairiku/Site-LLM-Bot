AIチャットボット MVP 開発進捗
更新日: 2026-04-27

現在の進行範囲
- 工程5「ウィジェットのブラッシュアップ」まで対応
- 販売用ウィジェット前提のサーバー設計書を追加

1. サンプルホームページ
- `demo/sample_page.html` を作成
- 住宅・リフォーム業向けのサンプル導線を意識したトップページ構成を実装
- ウィジェットを埋め込みやすいレイアウトに調整

2. 仮ウィジェット
- `static/mock-widget.js` を作成
- `static/mock-widget.css` を作成
- 右下固定ボタン型の仮ウィジェットを実装
- `demo/sample_page.html` へ埋め込み、モック応答での表示確認ができる構成にした

3. OpenAI API呼び出しハンドラ (ウィジェット -> API -> 回答生成の疎通確認を行うため、最低限の実装)
- `site_llm_bot/api/app.py` に FastAPI の最小APIを追加
- `POST /api/chat` で OpenAI Responses API を呼ぶ導線を実装
- APIキー未設定時はデモ応答へフォールバックする構成にした
- 仮ウィジェットから `data-api-base` 経由で API を呼べるように変更

4. ハンドラの最適化
- `site_llm_bot/config.py` にタイムアウト・履歴件数・許可Origin設定を追加
- `site_llm_bot/services/session_store.py` を追加し、オンメモリの会話セッション管理を実装
- `POST /api/chat` に Origin 検証、入力整形、`session_id` の入出力を追加
- OpenAI 呼び出しに直近履歴を渡せるようにし、会話継続の下地を整えた

5. ウィジェットのブラッシュアップ
- 閉じるボタン、送信中の入力制御、Enter送信、状態表示の改善を追加
- `/demo` と `/static/*` の組み合わせでローカル表示しやすい構成に調整
- API接続失敗時のモック応答フォールバックを維持したまま操作性を改善

6. Web Search 対応
- OpenAI Responses API の `web_search` ツールを使う構成へ更新
- `shintairiku.jp` を `allowed_domains` に設定し、回答前に対象サイトを検索対象にする形へ変更
- OpenAI 実疎通は未実施だが、送信 payload にドメイン制限付き Web Search 設定が入ることをテストで確認

7. テナント設定の外だし
- `config/tenants.json` を追加し、サンプルテナント設定をコード外へ移動
- `site_llm_bot/config.py` でテナント設定ファイルを読み込む構成へ変更
- `/api/chat` は `tenant_id` ごとに許可Origin・検索対象ドメインを参照するように変更
- sample page と widget から `tenant_id` を送る構成に調整

8. デプロイ準備
- `requirements.txt`、`Dockerfile`、`.dockerignore` を追加
- Vercel 用のルート `app.py` と `vercel.json` を追加
- `public/static/*` を追加し、Vercel の静的配信に対応
- `docs/デプロイ.md` に Vercel / Cloud Run の手順を整理

9. デプロイ先反映
- Vercel 確認用ドメインと Cloud Run ドメインを `allowed_origins` に追加
- sample page と widget の API 接続先既定値を Cloud Run へ変更
