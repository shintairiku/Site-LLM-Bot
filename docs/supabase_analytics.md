# Supabase 利用者数記録

## 作成するテーブルとRPC

Supabase Dashboard の SQL Editor で以下を実行します。

```sql
create table if not exists public.analytics_unique_users (
  id bigint generated by default as identity primary key,
  tenant_id text not null,
  visitor_id text not null,
  first_seen_at timestamptz not null default now(),
  origin text,
  page_url text,
  created_at timestamptz not null default now(),
  constraint analytics_unique_users_tenant_visitor_unique unique (tenant_id, visitor_id)
);

alter table public.analytics_unique_users enable row level security;

revoke all on table public.analytics_unique_users from anon, authenticated;

create or replace function public.record_unique_user(
  p_tenant_id text,
  p_visitor_id text,
  p_origin text default null,
  p_page_url text default null
)
returns boolean
language plpgsql
security definer
set search_path = public
as $$
declare
  inserted_count integer;
begin
  insert into public.analytics_unique_users (
    tenant_id,
    visitor_id,
    origin,
    page_url
  )
  values (
    p_tenant_id,
    p_visitor_id,
    p_origin,
    p_page_url
  )
  on conflict (tenant_id, visitor_id) do nothing;

  get diagnostics inserted_count = row_count;
  return inserted_count = 1;
end;
$$;

revoke all on function public.record_unique_user(text, text, text, text) from public;
grant execute on function public.record_unique_user(text, text, text, text) to service_role;
```

日次の利用者数は次のSQLで確認できます。

```sql
select
  tenant_id,
  date_trunc('day', first_seen_at at time zone 'Asia/Tokyo') as day_jst,
  count(*) as unique_users
from public.analytics_unique_users
group by tenant_id, day_jst
order by day_jst desc, tenant_id;
```

## Supabase の接続情報を取得する手順

1. Supabase Dashboard で対象プロジェクトを開く
2. Project Settings > API を開く
3. Project URL を `SUPABASE_URL` として控える
4. API Keys で Secret key、または Legacy service_role key を取得する
5. Cloud Run の環境変数に次を設定する

```text
SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=取得したSecret keyまたはservice_role key
```

`SUPABASE_SERVICE_ROLE_KEY` はサーバー専用です。ウィジェット、HTML、公開JSには絶対に埋め込まないでください。

補足:
- Secret key はSupabaseが推奨する新しいサーバー用キーです。
- Legacy service_role key も利用できます。
- この実装では `SUPABASE_SECRET_KEY` も `SUPABASE_SERVICE_ROLE_KEY` の代替として読み取ります。
- `sb_secret_...` 形式のSecret keyは `apikey` ヘッダーで送信します。
- Legacy `service_role` JWTの場合のみ `Authorization: Bearer ...` も付与します。
