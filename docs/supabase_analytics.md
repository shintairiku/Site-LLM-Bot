# Supabase 分析データ記録

## 利用者数記録のテーブルとRPC

Supabase Dashboard の SQL Editor で以下を実行します。

```sql
create table if not exists public.analytics_unique_users (
  id bigint generated by default as identity primary key,
  tenant_id text not null,
  visitor_id text not null,
  first_seen_at timestamptz not null default now(),
  origin text,
  page_url text,
  created_at timestamptz not null default now(),
  constraint analytics_unique_users_tenant_visitor_unique unique (tenant_id, visitor_id)
);

alter table public.analytics_unique_users enable row level security;

revoke all on table public.analytics_unique_users from anon, authenticated;

create or replace function public.record_unique_user(
  p_tenant_id text,
  p_visitor_id text,
  p_origin text default null,
  p_page_url text default null
)
returns boolean
language plpgsql
security definer
set search_path = public
as $$
declare
  inserted_count integer;
begin
  insert into public.analytics_unique_users (
    tenant_id,
    visitor_id,
    origin,
    page_url
  )
  values (
    p_tenant_id,
    p_visitor_id,
    p_origin,
    p_page_url
  )
  on conflict (tenant_id, visitor_id) do nothing;

  get diagnostics inserted_count = row_count;
  return inserted_count = 1;
end;
$$;

revoke all on function public.record_unique_user(text, text, text, text) from public;
grant execute on function public.record_unique_user(text, text, text, text) to service_role;
```

日次の利用者数は次のSQLで確認できます。

```sql
select
  tenant_id,
  date_trunc('day', first_seen_at at time zone 'Asia/Tokyo') as day_jst,
  count(*) as unique_users
from public.analytics_unique_users
group by tenant_id, day_jst
order by day_jst desc, tenant_id;
```

## チャット送信回数記録のテーブルとRPC

Supabase Dashboard の SQL Editor で以下を実行します。

**新規作成の場合:**

```sql
create table if not exists public.analytics_chat_messages (
  id bigint generated by default as identity primary key,
  tenant_id text not null,
  session_id text not null,
  visitor_id text,
  sent_at timestamptz not null,
  origin text,
  page_url text,
  question_text text,
  created_at timestamptz not null default now()
);

create index if not exists analytics_chat_messages_tenant_sent_at_idx
  on public.analytics_chat_messages (tenant_id, sent_at);

alter table public.analytics_chat_messages enable row level security;

revoke all on table public.analytics_chat_messages from anon, authenticated;
```

**既存テーブルへのカラム追加（データが既にある場合）:**

```sql
alter table public.analytics_chat_messages
  add column if not exists question_text text;
```

**RPC 関数の作成・更新:**

```sql
create or replace function public.record_chat_message_sent(
  p_tenant_id text,
  p_session_id text,
  p_visitor_id text default null,
  p_origin text default null,
  p_page_url text default null,
  p_occurred_at timestamptz default now(),
  p_question_text text default null
)
returns void
language plpgsql
security definer
set search_path = public
as $$
begin
  insert into public.analytics_chat_messages (
    tenant_id,
    session_id,
    visitor_id,
    origin,
    page_url,
    sent_at,
    question_text
  )
  values (
    p_tenant_id,
    p_session_id,
    p_visitor_id,
    p_origin,
    p_page_url,
    coalesce(p_occurred_at, now()),
    p_question_text
  );
end;
$$;

revoke all on function public.record_chat_message_sent(
  text,
  text,
  text,
  text,
  text,
  timestamptz,
  text
) from public;
grant execute on function public.record_chat_message_sent(
  text,
  text,
  text,
  text,
  text,
  timestamptz,
  text
) to service_role;
```

> `question_text` はアプリ側でメールアドレス・電話番号・郵便番号・クレジットカード番号をマスクしてから記録します。

日次・週次・月次のチャット送信回数は次のSQLで確認できます。

```sql
select
  tenant_id,
  date_trunc('day', sent_at at time zone 'Asia/Tokyo') as day_jst,
  count(*) as chat_message_count
from public.analytics_chat_messages
group by tenant_id, day_jst
order by day_jst desc, tenant_id;
```

```sql
select
  tenant_id,
  date_trunc('week', sent_at at time zone 'Asia/Tokyo') as week_jst,
  count(*) as chat_message_count
from public.analytics_chat_messages
group by tenant_id, week_jst
order by week_jst desc, tenant_id;
```

```sql
select
  tenant_id,
  date_trunc('month', sent_at at time zone 'Asia/Tokyo') as month_jst,
  count(*) as chat_message_count
from public.analytics_chat_messages
group by tenant_id, month_jst
order by month_jst desc, tenant_id;
```

テナント・リンクURLごとの合計クリック回数は次のSQLで確認できます。

```sql
select
  tenant_id,
  link_url,
  count(*) as click_count
from public.analytics_related_link_clicks
group by tenant_id, link_url
order by tenant_id, click_count desc, link_url;
```

## Supabase の接続情報を取得する手順

1. Supabase Dashboard で対象プロジェクトを開く
2. Project Settings > API を開く
3. Project URL を `SUPABASE_URL` として控える
4. API Keys で Secret key、または Legacy service_role key を取得する
5. Cloud Run の環境変数に次を設定する

```text
SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=取得したSecret keyまたはservice_role key
```

`SUPABASE_SERVICE_ROLE_KEY` はサーバー専用です。ウィジェット、HTML、公開JSには絶対に埋め込まないでください。

補足:
- Secret key はSupabaseが推奨する新しいサーバー用キーです。
- Legacy service_role key も利用できます。
- この実装では `SUPABASE_SECRET_KEY` も `SUPABASE_SERVICE_ROLE_KEY` の代替として読み取ります。
- `sb_secret_...` 形式のSecret keyは `apikey` ヘッダーで送信します。
- Legacy `service_role` JWTの場合のみ `Authorization: Bearer ...` も付与します。
