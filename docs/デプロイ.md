# デプロイ手順

## Vercel

前提:
- ルートの `app.py` を Vercel の FastAPI エントリポイントとして使う
- `public/static/*` は Vercel の静的配信に使う

必要な環境変数:
- `OPENAI_API_KEY`
- 必要なら `OPENAI_MODEL`
- 必要なら `TENANT_CONFIG_PATH`

手順:
1. Git リポジトリを Vercel に接続
2. 環境変数を設定
3. デプロイ

確認先:
- `/`
- `/demo`
- `/health`

## Cloud Run

前提:
- `Dockerfile` を使ってコンテナデプロイする
- Cloud Run は `PORT` 環境変数を自動注入する

必要な環境変数:
- `OPENAI_API_KEY`
- 必要なら `OPENAI_MODEL`

例:

```bash
gcloud run deploy site-llm-bot \
  --source . \
  --region asia-northeast1 \
  --allow-unauthenticated
```

またはコンテナビルド後にデプロイ:

```bash
gcloud builds submit --tag gcr.io/PROJECT_ID/site-llm-bot
gcloud run deploy site-llm-bot \
  --image gcr.io/PROJECT_ID/site-llm-bot \
  --region asia-northeast1 \
  --allow-unauthenticated
```
