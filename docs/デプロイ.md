# デプロイ手順

## Vercel

前提:
- ルートの `app.py` を Vercel の FastAPI エントリポイントとして使う
- `public/static/*` は Vercel の静的配信に使う

必要な環境変数:
- `OPENAI_API_KEY`
- 必要なら `OPENAI_MODEL`
  既定値は `gpt-5.4-mini`
- 必要なら `TENANT_CONFIG_PATH`
- Preview から開発用 Cloud Run を呼ぶ場合は `WIDGET_API_BASE`
  例: `https://dev-site-llm-bot-xxxxx.a.run.app`

動作確認用の接続先:
- フロント確認ページ例: `https://site-llm-rkhxjn6ty-marketing-automation.vercel.app/`
- API 例: `https://site-llm-bot-742231208085.asia-northeast1.run.app`

手順:
1. Git リポジトリを Vercel に接続
2. 環境変数を設定
3. デプロイ

確認先:
- `/`
- `/demo`
- `/health`

## 販売先へ渡す script タグ

Cloud Run サービスエンドポイント URL を固定配信元として使う。

```html
<script
  id="site-llm-bot-widget"
  src="https://site-llm-bot-742231208085.asia-northeast1.run.app/static/widget.js"
  data-tenant-id="顧客別tenant-id"
  data-tenant-name="顧客名"
  data-public-token="顧客別public-token"
  data-api-base="https://site-llm-bot-742231208085.asia-northeast1.run.app"
  defer
></script>
```

固定 URL から `widget.js`、`widget.css`、`tenants/{tenant_id}.css` を配信するため、販売先へ渡すものは基本的に script タグだけでよい。

## Cloud Run

前提:
- `Dockerfile` を使ってコンテナデプロイする
- Cloud Run は `PORT` 環境変数を自動注入する

必要な環境変数:
- `OPENAI_API_KEY`
- 必要なら `OPENAI_MODEL`
  既定値は `gpt-5.4-mini`
- 必要なら `WIDGET_API_BASE`
  未設定の場合、サンプルページの配信元と同じオリジンの API を使う

CORS:
- `config/tenants.json` の `allowed_origins` に Vercel 側の確認用ドメインを追加しておく

例:

```bash
gcloud run deploy site-llm-bot \
  --source . \
  --region asia-northeast1 \
  --allow-unauthenticated
```

またはコンテナビルド後にデプロイ:

```bash
gcloud builds submit --tag gcr.io/PROJECT_ID/site-llm-bot
gcloud run deploy site-llm-bot \
  --image gcr.io/PROJECT_ID/site-llm-bot \
  --region asia-northeast1 \
  --allow-unauthenticated
```
